Web Admin
Dashboard

Tài liệu yêu cầu sản phẩm cho hệ thống quản trị (Back-office) phục vụ vận hành nội dung POI, Tour và Narration cho App Mobile Vĩnh Khánh Tourism thông qua các REST API hiện có.

§ 01

Overview & Goals

Hiện tại, dữ liệu POI (điểm tham quan) trong hệ thống Vĩnh Khánh Tourism chỉ có thể được quản lý trực tiếp qua database hoặc Swagger UI. Điều này tạo rủi ro cao (không có validation, không audit log) và không thể giao cho nhân viên vận hành phi kỹ thuật. Web Admin Dashboard là giải pháp back-office cho phép Admin quản lý toàn bộ nội dung qua giao diện thân thiện, tương tác trực tiếp với các REST API hiện có.

Mục tiêu vận hành

Cho phép Admin thêm, sửa, xóa POI và Tour mà không cần can thiệp kỹ thuật vào database hoặc code.

Mục tiêu nội dung

Quản lý đa ngôn ngữ (Narration), ảnh Cloudinary, Geofence và QR Code cho từng điểm tham quan.

Mục tiêu an toàn dữ liệu

Mọi thay đổi phải qua xác thực JWT với phân quyền Admin/Manager. Không có action destructive nào được thực thi mà không có confirmation step.

Mục tiêu tốc độ triển khai

MVP tập trung tối đa vào các endpoint API đã có sẵn để rút ngắn thời gian dev. Không build thêm API mới trong Sprint 1.

Phạm vi MVP (In-scope)

  • Module Auth: Login / Logout với JWT
  • Module POI: CRUD + gán tọa độ trên bản đồ + phân loại Major/Minor
  • Module POI Images: Upload đơn/nhiều ảnh, reorder, xóa (Cloudinary)
  • Module Narrations: CRUD narration text, upload audio file, switch TTS/Audio
  • Module Restaurant & Food: CRUD nhà hàng và menu món ăn theo POI
  • Module Geofence: Cấu hình bán kính geofence cho từng POI
  • Module QR Code: Tạo và quản lý QR Code cho POI
  • Module Language: Thêm/xóa ngôn ngữ hỗ trợ
  • Module Tour: Tạo tour bằng cách chọn và sắp xếp các POI có sẵn
Out of scope (v1.0)

Analytics dashboard, user management của Tourist, push notification, export báo cáo, multi-tenant, và dark mode. Sẽ xem xét tại v2.0.

§ 02

User Personas

A
Anh Minh — Admin Hệ Thống
Role: Admin

Quản trị viên kỹ thuật, có quyền toàn bộ hệ thống. Chịu trách nhiệm cấu hình ngôn ngữ, xóa dữ liệu nhạy cảm, và quản lý tài khoản Manager. Sử dụng Web Admin hằng ngày.

Toàn quyền CRUD Xóa ngôn ngữ Quản lý Geofence Tạo QR Code
M
Chị Lan — Content Manager
Role: Manager

Nhân viên vận hành nội dung, không có nền tảng kỹ thuật. Chịu trách nhiệm cập nhật thông tin POI, upload ảnh và audio, tạo Tour. Không có quyền xóa ngôn ngữ hoặc xóa POI đang trong tour đang hoạt động.

Thêm/Sửa POI Upload ảnh & audio Tạo & sửa Tour Quản lý Restaurant
Phân quyền API hiện có

Backend LanguageController đã implement [Authorize(Roles = "Admin,Manager")] cho POST và [Authorize(Roles = "Admin")] cho DELETE. Web Admin cần phản ánh đúng phân quyền này trên UI (ẩn/vô hiệu hóa nút theo role từ JWT payload).

§ 03

User Stories

Module AUTH
US-01 Là Admin/Manager, tôi muốn đăng nhập bằng username và password để truy cập vào hệ thống quản trị một cách bảo mật. P0
US-02 Là Admin/Manager, tôi muốn hệ thống tự động đăng xuất sau 24 giờ để bảo vệ tài khoản khi quên đăng xuất. P0
US-03 Là Admin/Manager, tôi muốn thấy rõ tên và role của mình trên header để biết mình đang thao tác với quyền hạn nào. P1
Module POI
US-04 Là Admin, tôi muốn thêm một POI mới bằng cách click trực tiếp lên bản đồ để lấy tọa độ thay vì nhập tay latitude/longitude. P0
US-05 Là Admin, tôi muốn phân loại POI là Major (địa điểm chính) hoặc Minor (WC / Bán vé / Gửi xe / Bến thuyền) để App Mobile hiển thị đúng icon và ưu tiên. P0
US-06 Là Manager, tôi muốn xem danh sách tất cả POI kèm ảnh thumbnail, đánh giá trung bình và số lượt review để nhanh chóng nắm tình trạng nội dung. P0
US-07 Là Manager, tôi muốn upload nhiều ảnh cùng lúc cho một POI và kéo thả để sắp xếp thứ tự hiển thị, ảnh đầu tiên sẽ là ảnh bìa. P0
US-08 Là Admin, tôi muốn cấu hình Geofence (bán kính tính bằng mét) cho từng POI để điều chỉnh vùng trigger tự động thuyết minh trên App Mobile. P1
US-09 Là Admin, tôi muốn tạo QR Code cho POI để khách scan bằng App và truy cập chi tiết địa điểm. P1
Module NARRATION
US-10 Là Manager, tôi muốn nhập bài thuyết minh (text) theo từng ngôn ngữ cho mỗi POI để hỗ trợ khách quốc tế. P0
US-11 Là Manager, tôi muốn upload file audio (MP3/WAV) thay thế cho TTS để nội dung thuyết minh có giọng đọc chuyên nghiệp hơn. P1
US-12 Là Manager, tôi muốn toggle switch giữa chế độ "Dùng TTS" và "Dùng Audio file" trên từng narration để linh hoạt A/B test. P1
Module RESTAURANT & FOOD
US-13 Là Manager, tôi muốn thêm nhà hàng/quán ăn vào POI và nhập menu với giá để khách xem trước khi đến. P1
US-14 Là Manager, tôi muốn cập nhật giá món ăn khi có thay đổi mà không cần sửa toàn bộ POI. P1
Module TOUR
US-15 Là Admin, tôi muốn tạo Tour bằng cách chọn POIs từ danh sách có sẵn và kéo thả để sắp xếp thứ tự di chuyển, hệ thống tự cập nhật lộ trình trên bản đồ. P0
US-16 Là Admin, tôi muốn hệ thống không cho tạo Tour nếu chưa có ít nhất 2 POI, đồng thời hiển thị lỗi rõ ràng. P0
US-17 Là Manager, tôi muốn xem preview lộ trình Tour trên bản đồ với đường nối giữa các POI theo đúng thứ tự đã đặt. P1
§ 04

Screens & UI

🔐
Login Page /login
  • Form gồm 2 trường: Username và Password (masked). Nút "Đăng nhập" full-width.
  • Khi submit: gọi POST /api/User/login, nhận JWT, lưu vào localStorage dưới key admin_jwt.
  • Parse JWT payload để lấy ClaimTypes.Role. Nếu role là Tourist, hiển thị lỗi "Tài khoản không có quyền truy cập Admin".
  • Redirect về /dashboard sau khi đăng nhập thành công.
  • Route này là public duy nhất — mọi route khác đều require authenticated JWT.
📊
Dashboard Overview /dashboard
  • Stat cards: Tổng số POI | POI có Geofence | POI có Audio | Tổng Tour.
  • Bản đồ mini hiển thị toàn bộ POI với pin màu khác nhau theo loại (Major/Minor).
  • Quick action buttons: "Thêm POI mới", "Tạo Tour", "Upload Audio".
  • Activity feed: 10 action gần nhất (nếu có audit log backend — xem Missing API §11).
📍
POI Management — List View /pois
  • Bảng dữ liệu (data table) với cột: Thumbnail ảnh đầu tiên | Tên POI | Loại (Major/Minor + subtype) | Lat/Lng | Rating ⭐ | Review count | Geofence (✓/✗) | QR (✓/✗) | Actions.
  • Filter thanh bên trái: Theo loại POI (Major / WC / Bán vé / Gửi xe / Bến thuyền).
  • Tìm kiếm real-time theo tên POI.
  • Nút toggle View: chuyển đổi giữa List View và Map View.
  • Nút "Thêm POI" ở header — mở modal hoặc navigate sang /pois/new.
  • Click vào row POI để mở trang POI Detail (/pois/:id).
  • Nút xóa trong row: hiện confirmation dialog trước khi gọi DELETE.
🗺️
POI Management — Map View /pois?view=map
  • Bản đồ full-screen (Leaflet.js hoặc Mapbox GL) hiển thị tất cả POI dưới dạng marker.
  • Marker color: Xanh lá = Major, Đỏ = Minor (WC), Vàng = Bán vé, Xanh dương = Gửi xe, Tím = Bến thuyền.
  • Click vào vị trí trống trên bản đồ: mở sidebar form "Tạo POI mới" với Lat/Lng đã tự điền từ click point. Admin chỉ cần nhập Tên và Loại.
  • Click vào marker POI đã có: mở popup với tên, rating, nút "Edit" và "Xóa".
  • Kéo marker để cập nhật vị trí — hiện confirmation "Cập nhật tọa độ POI này?".
  • Hiển thị vòng tròn Geofence (circle overlay) xung quanh POI có Geofence, bán kính theo dữ liệu DB.
✏️
POI Detail & Edit Page /pois/:id
  • Thông tin cơ bản: Form edit tên, loại POI, mô tả, lat/lng (có mini-map để xem vị trí).
  • Tab Ảnh: Grid ảnh từ Cloudinary, kéo thả để reorder, nút upload (đơn và batch), nút xóa từng ảnh (gọi DELETE + xóa trên Cloudinary qua PublicId).
  • Tab Narrations: Bảng narration theo ngôn ngữ. Mỗi row có: Language Code | Text preview | Toggle TTS/Audio | Upload audio button | Preview player. Nút "Thêm narration" mở form chọn ngôn ngữ + nhập text.
  • Tab Restaurant: Danh sách nhà hàng của POI. Expand từng nhà hàng để xem/edit menu món ăn.
  • Tab Geofence: Form nhập bán kính (meters) + mini-map hiển thị vòng tròn preview radius. Nếu chưa có geofence thì hiện "Tạo Geofence". Nếu đã có thì edit bán kính.
  • Tab QR Code: Hiển thị QR image preview (render từ CodeValue URL), nút "Tạo QR" nếu chưa có, nút Download PNG.
🛤️
Tour Builder /tours/new · /tours/:id/edit
  • Layout 3 cột: [Danh sách POI khả dụng] | [Builder — danh sách đã chọn + drag handle] | [Preview bản đồ lộ trình]
  • Cột 1 — POI Pool: Tìm kiếm POI theo tên. Filter theo loại. Card từng POI với ảnh thumbnail, tên, rating. Nút "+" để thêm vào tour.
  • Cột 2 — Tour Builder: Drag-and-drop list (dùng thư viện @dnd-kit/sortable hoặc react-beautiful-dnd). Mỗi item hiển thị số thứ tự (1, 2, 3…), tên POI, thumbnail, nút xóa khỏi tour. Sau mỗi lần reorder: tự động cập nhật polyline trên cột 3.
  • Cột 3 — Map Preview: Vẽ polyline nối các POI theo thứ tự. Đánh số thứ tự trên marker. Cập nhật realtime khi kéo thả ở cột 2.
  • Form meta: Tên Tour, Mô tả, Ngôn ngữ, Thời lượng ước tính.
  • Validation: Tối thiểu 2 POI mới cho phép Save. Tên Tour không được để trống.
🌐
Language Management /languages
  • Bảng danh sách ngôn ngữ: Language Code | Tên đầy đủ | Số Narration đang dùng.
  • Form thêm ngôn ngữ: Language Code (vi, en, zh…) + Tên hiển thị. Chỉ Manager và Admin mới thấy nút thêm.
  • Nút xóa: Chỉ Admin mới thấy. Cần confirmation: "Xóa ngôn ngữ này sẽ ảnh hưởng đến X narration đang dùng. Tiếp tục?"

§ 06

State Management

Mọi component có call API phải xử lý đầy đủ 4 trạng thái sau:

Default Loading Empty Error Success
State Trigger UI hiển thị Action
Loading API đang gọi Skeleton loader (không dùng spinner tròn) Disable nút save/submit
Empty API trả về [] rỗng Illustration + message + nút CTA (vd: "Thêm POI đầu tiên") Link đến action tương ứng
Error API trả về 4xx/5xx hoặc network error Banner đỏ trên cùng với error message từ response body Nút "Thử lại"
Success API trả về 200/201/204 Toast notification 3 giây rồi tự dismiss Refresh data trong background

Validation Rules (Client-side)

FieldRuleMessage lỗi
POI Name Required, 3–100 ký tự Tên POI phải từ 3–100 ký tự
Latitude Required, số thực, -90 đến 90 Vĩ độ không hợp lệ
Longitude Required, số thực, -180 đến 180 Kinh độ không hợp lệ
Geofence Radius Required, số nguyên, 10–5000 (mét) Bán kính phải từ 10 đến 5000 mét
Audio File Extension: .mp3 .wav .ogg .m4a .aac. Size ≤ 50MB File phải là định dạng âm thanh, tối đa 50MB
Image File Extension: .jpg .jpeg .png .webp .gif. Size ≤ 10MB File phải là hình ảnh, tối đa 10MB
Tour POIs Tối thiểu 2 POI Tour phải có ít nhất 2 điểm tham quan
Food Price Required, số dương, tối đa 10 triệu Giá món ăn không hợp lệ

§ 08

Acceptance Criteria

AC-01: Đăng nhập Admin
GivenAdmin truy cập trang /login
WhenNhập đúng username và password của tài khoản có role "Admin" hoặc "Manager", nhấn Đăng nhập
ThenHệ thống lưu JWT vào localStorage, redirect về /dashboard, header hiển thị tên và role của user
AC-02: Từ chối Tourist đăng nhập Admin
GivenNgười dùng có role "Tourist" nhập đúng thông tin đăng nhập
WhenNhấn Đăng nhập, API trả về JWT thành công
ThenWeb Admin parse JWT, phát hiện role = "Tourist", hiển thị thông báo lỗi "Tài khoản không có quyền truy cập Admin", KHÔNG lưu token, KHÔNG redirect
AC-03: Tạo POI mới bằng click bản đồ
GivenAdmin đang ở Map View (/pois?view=map) và đã đăng nhập
WhenAdmin click vào vị trí trống trên bản đồ, nhập Tên POI và chọn Loại, nhấn "Tạo POI"
ThenHệ thống gọi POST /api/POIs với Lat/Lng từ click point. Marker mới xuất hiện trên bản đồ. Toast "Tạo POI thành công". Redirect sang /pois/:newId để tiếp tục nhập nội dung
AC-04: Kéo thả reorder ảnh POI
GivenPOI có ít nhất 2 ảnh trong Tab Ảnh
WhenAdmin kéo ảnh thứ 3 lên vị trí thứ 1 và thả
ThenUI cập nhật thứ tự ngay lập tức (optimistic). Hệ thống gọi PATCH /api/PoiImages/reorder với DisplayOrder mới. Nếu API thành công: toast xanh. Nếu thất bại: roll back về thứ tự cũ + toast đỏ
AC-05: Upload audio và switch sang Audio mode
GivenNarration đang ở chế độ TTS (UseAudioFile = false)
WhenManager upload file .mp3 ≤ 50MB, sau đó toggle switch sang "Dùng Audio"
ThenHệ thống gọi POST /api/Narrations/:id/audio → file lên Cloudinary → URL được lưu. Sau đó gọi PATCH /api/Narrations/:id/use-audio. App Mobile lần sau vào vùng geofence sẽ phát AudioUrl thay vì TTS
AC-06: Tạo Tour với kéo thả sắp xếp POI
GivenAdmin đang ở Tour Builder với ít nhất 3 POI đã được thêm vào danh sách tour
WhenAdmin kéo POI thứ 3 lên vị trí thứ 1, sau đó nhấn Save
ThenPolyline trên Map Preview cập nhật realtime khi kéo thả. Khi Save: gọi POST /api/Tour với mảng TourPOIs có SortOrder đúng thứ tự mới. Response trả về TourId mới. Redirect sang /tours/:id
AC-07: Validation Tour tối thiểu 2 POI
GivenAdmin ở Tour Builder và chỉ mới thêm 1 POI vào danh sách
WhenAdmin nhấn nút "Save Tour"
ThenNút Save bị disable. Hiển thị inline message "Tour cần ít nhất 2 điểm tham quan". KHÔNG gọi API. Nút chỉ enable khi danh sách có ≥ 2 POI và tên tour không rỗng
AC-08: Phân quyền xóa ngôn ngữ
GivenUser đăng nhập với role "Manager" (không phải "Admin")
WhenUser mở trang Language Management
ThenNút "Xóa ngôn ngữ" bị ẩn hoàn toàn (không chỉ disabled). Nút "Thêm ngôn ngữ" vẫn hiển thị vì Manager có quyền POST. Nguyên tắc: UI phải phản ánh đúng phân quyền từ role trong JWT
§ 09

Non-Functional Requirements

Bảo mật
  • JWT lưu trong localStorage hoặc httpOnly cookie (cookie an toàn hơn)
  • Tất cả API call phải gắn header Authorization: Bearer <token>
  • Interceptor tự động redirect về /login khi nhận 401
  • Không log JWT token ra console
  • Confirm dialog bắt buộc với mọi action xóa dữ liệu
  • JWT expire 24h (theo backend config) — Web Admin phải handle gracefully
Hiệu năng
  • Danh sách POI load ≤ 2 giây trên kết nối 4G bình thường
  • Bản đồ render POI markers ≤ 3 giây với ≤ 500 markers
  • Upload ảnh batch hiển thị progress bar % realtime
  • Debounce tìm kiếm 300ms để không spam API
  • Lazy load ảnh trong grid (Intersection Observer)
  • Paginate danh sách POI nếu > 50 items
Usability
  • Tất cả form có auto-save draft (localStorage) để tránh mất data khi F5
  • Breadcrumb navigation rõ ràng: Dashboard › POIs › Tên POI
  • Responsive: hoạt động tốt trên màn hình 1280px trở lên
  • Keyboard shortcuts: Ctrl+S để save, Esc để đóng modal
  • Undo action cho xóa (soft-undo trong 5 giây qua toast)
Validation & Data Integrity
  • Validation chạy cả client-side (UX) và server-side (source of truth)
  • Không cho phép xóa POI nếu POI đó đang được dùng trong Tour đang active
  • File upload: validate type và size trước khi gọi API
  • Geofence: mỗi POI chỉ có 1 geofence (backend đã enforce, UI phải reflect)
  • Language code: unique constraint (backend đã enforce)

§ 11

Missing APIs

Lưu ý cho Backend team

Các API sau đây chưa tồn tại trong codebase. Cần implement thêm trước khi Web Admin Sprint 1 kết thúc. Ưu tiên theo P0 trước.

🔴 [P0] POST /api/POIs — Tạo POI mới
POIsController hiện chỉ có GET (GetAll và GetById). Hoàn toàn thiếu endpoint để tạo, sửa, xóa POI. Đây là tính năng cốt lõi nhất của Web Admin.
Cần thêm: POST /api/POIs · PUT /api/POIs/{id} · DELETE /api/POIs/{id}
🔴 [P0] Tour CRUD — Toàn bộ Tour API
Không có bất kỳ controller nào liên quan đến Tour trong codebase hiện tại. Cần tạo TourController mới với đầy đủ CRUD. Model cần có: Tour entity và TourPOI (bảng nối có field SortOrder).
Cần tạo: GET /api/Tour · POST /api/Tour · PUT /api/Tour/{id} · DELETE /api/Tour/{id} · PUT /api/Tour/{id}/reorder-pois
🟡 [P1] GET /api/Geofence/{poiId} — Lấy Geofence theo POI
GeofenceController chỉ có GET /api/Geofence (lấy toàn bộ). Web Admin cần truy vấn geofence của 1 POI cụ thể để hiển thị trong Tab Geofence. Hiện tại phải filter client-side từ danh sách đầy đủ — không hiệu quả.
Cần thêm: GET /api/Geofence/ByPOI/{poiId}
🟡 [P1] DELETE /api/Geofence/{id} — Xóa Geofence
Không có endpoint DELETE cho Geofence. Admin không thể xóa geofence và tạo lại nếu cần reset. Hiện chỉ có PUT để sửa bán kính.
Cần thêm: DELETE /api/Geofence/{id}
🔵 [P2] Audit Log API — Lịch sử hoạt động
Dashboard muốn hiển thị "10 action gần nhất" nhưng backend chưa có audit log. Nếu không implement, bỏ phần này khỏi Dashboard MVP.
Optional: GET /api/AuditLog?limit=10
🔵 [P2] GET /api/User/me (với DisplayName)
API GET /api/User/me hiện trả về username và role nhưng không có DisplayName. Web Admin header muốn hiển thị tên thân thiện. Có thể bổ sung DisplayName vào response.
Sửa GET /api/User/me response thêm field DisplayName
§ 12

Open Questions & Assumptions

  • Q1
    POI Type (Major/Minor) được lưu ở đâu trong DB?
    Giả định: Hiện chưa có field PoiType trong model POI (code chưa thể hiện). Dev Backend cần thêm field PoiType (enum hoặc string) vào bảng POI và expose ra GET/POST response. Web Admin dùng field này để render màu marker trên bản đồ.
  • Q2
    Xóa POI có cascade xóa Geofence, QR Code, Narration, và ảnh Cloudinary không?
    Giả định: Backend phải xử lý cascade delete (xóa PoiImages trên Cloudinary, xóa Audio trên Cloudinary trước khi xóa record). Web Admin chỉ gọi DELETE /api/POIs/{id} và không chịu trách nhiệm cleanup. Cần Backend team confirm.
  • Q3
    Review hiện tại là mock data — khi nào API review thực thật được implement?
    Giả định: Tab Review trong DetailScreen.xaml.cs dùng mock data. Web Admin v1.0 chỉ hiển thị ReadOnly review list từ POIDto.Reviews. Chức năng phê duyệt/xóa review sẽ là v2.0 sau khi có Review API thật.
  • Q4
    Bản đồ nào được dùng cho Web Admin — Google Maps hay OpenStreetMap?
    Giả định: Mobile App dùng Microsoft.Maui.Controls.Maps (OpenStreetMap/Google Maps tùy platform). Web Admin đề xuất dùng Leaflet.js + OpenStreetMap (miễn phí, không cần API key billing) để phù hợp budget MVP. Nếu cần Google Maps, phải request API key và xác nhận billing.
  • Q5
    Tour có hiển thị trên App Mobile ngay khi tạo không, hay cần "Publish" thủ công?
    Giả định: Do Tour API chưa có, chưa rõ flow. Đề xuất thêm field IsPublished (bool) vào Tour model để Admin có thể tạo Tour ở chế độ Draft → Preview → Publish khi đã sẵn sàng. Cần confirm với Product Owner.
  • Q6
    Khi tạo QR Code — CodeValue URL dùng domain thật hay placeholder?
    Giả định: Code backend hiện dùng hardcode https://myapp.com/poi/{id}. Cần config BaseAppUrl trong appsettings.json và Web Admin cho phép Admin preview QR image (render từ URL) trước khi in ấn.
  • Q7
    Dữ liệu MapPage hiện là hardcode — khi nào chuyển sang API?
    Giả định: MapPage.xaml.cs đang dùng danh sách LoadVinhKhanhRestaurants() hardcode. Sau khi Web Admin có thể tạo POI qua API, App Mobile cần refactor để gọi GET /api/POIs thay vì hardcode. Web Admin tạo data thật → App Mobile consume. Đây là dependency cần align với Mobile team.
  • Q8
    Có cần log audit khi Admin xóa POI hoặc thay đổi Geofence không?
    Giả định: Không yêu cầu trong v1.0 MVP. Nếu cần, sẽ implement Audit Log middleware ở v2.0. Hiện tại chỉ dùng confirmation dialog như lớp bảo vệ chính.
Bước tiếp theo

Team Backend ưu tiên implement POST/PUT/DELETE /api/POIs và Tour CRUD API (§11, P0) trước khi Frontend Web Admin bắt đầu Sprint 1. Frontend có thể bắt đầu với Login, Dashboard shell, và POI List/Map View dùng GET APIs đã có sẵn trong khi chờ các API còn thiếu.
